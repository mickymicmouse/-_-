3. 데이터 입,출력 구현
데이터 모델 구성요소
	개체
	속성
	관계
종류
	개념적 데이터 모델(정보 모델) - 속성으로 기술된 개체 타입 + 개체 타입들 간의 관계
		E-R 모델
	논리적 데이터 모델(데이터 모델) - 필드로 기술된 데이터 타입 + 데이터 타입들 간의 관계
		관계 모델
		계층 모델
		네트워크 모델
	물리적 데이터 모델(물리 데이터 베이스 설계 과정)
요소
	구조
	연산
	제약 조건
이상
	삽입
	삭제
	갱신
종속
	완전 함수적 종속
	부분 함수적 종속
정규화
	1정규화 - 모든 속성의 도메인이 원자값을 갖도록 하는 것
	2정규화 - 기본키가 아닌 모든 속성이 기본키에 대해 완전 함수적 종속을 갖는 것
	3정규화 - 기본키가 아닌 모든 속성이 이행적 함수적 종속을 갖지 않는 것
		이행적 함수적 종속 - A->B, B->C A->C는 이행적 함수적 종속, 그러므로 분할 해야 됨. 이것이 제 3정규화
	BCNF - 모든 결정자가 후보키가 되도록 하는 것, 일반적으로 제 3정규형에 후보키가 여러개 존재하고, 이러한 후보키들이 서로 중첩되어 나타나는 경우에 적용 가능
	4정규화 - 다중값 종속
	5정규화 - 조인 종속

논리 데이터 모델 -> 물리 데이터 모델
